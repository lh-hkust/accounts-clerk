# Hermes - 卡片手势设计规范

> 定义卡片组件的手势交互模式和视觉反馈规范

---

## 一、手势层级定义

### 1.1 手势类型

| 手势 | 交互层级 | 用户心智 |
|------|----------|----------|
| **点击** | 详情查看 | 查看更多信息 |
| **长按** | 快捷菜单 | 快捷操作入口 |
| **右滑** | 最高频操作 | 最常用的操作 |
| **左滑** | 次要操作 | 辅助操作 |

### 1.2 手势优先级

```
右滑 > 左滑 > 长按 > 点击

规则：
- 滑动手势优先触发（覆盖点击）
- 长按需要持续按压500ms触发
- 点击为瞬间释放触发
```

### 1.3 手势互斥

| 手势组合 | 触发行为 |
|----------|----------|
| 右滑 + 点击 | 右滑触发 |
| 左滑 + 点击 | 左滑触发 |
| 长按 + 点击 | 长按触发（500ms后） |
| 滑动 + 长按 | 滑动触发 |

## 二、滑动手势视觉反馈

### 2.1 右滑视觉反馈

```
背景：主色 (#3b82f6) 渐变，从右侧向左展开
按钮：白色文字，16sp，粗体
阈值：滑动超过卡片宽度30%触发按钮显示
```

### 2.2 左滑视觉反馈

```
背景：危险色 (#ef4444) 或次要色渐变，从左侧向右展开
按钮：白色文字，16sp，粗体
阈值：滑动超过卡片宽度30%触发按钮显示
```

### 2.3 滑动阶段定义

| 阶段 | 滑动距离 | 行为 |
|------|----------|------|
| 开始滑动 | 0-30% | 背景渐变显示 |
| 超过阈值 | 30%+ | 按钮完全显示，可点击 |
| 深度滑动 | 50%+ | 可选自动触发操作 |
| 释放手指 | 任意 | 根据位置决定是否触发 |

## 三、长按菜单规范

### 3.1 菜单样式

| 属性 | 值 |
|------|------|
| 类型 | DropdownMenu 或 ModalBottomSheet |
| 位置 | 卡片下方或屏幕底部 |
| 动画 | 淡入淡出 + 滑动效果 |
| 菜单项高度 | 48dp |
| 文字对齐 | 左对齐 |
| 分隔线 | 1dp，边框色 (#334155) |

### 3.2 菜单项动态显示

菜单项根据状态动态显示可用操作，不可用操作不显示（而非禁用）。

## 四、技术实现方案

### 4.1 Jetpack Compose API

| 手势 | API |
|------|-----|
| 点击 | `Modifier.clickable { }` |
| 长按 | `Modifier.combinedClickable(onLongClick = { })` |
| 滑动 | `SwipeToDismiss` 或自定义手势检测 |

### 4.2 滑动实现要点

使用 `rememberDismissState` 管理滑动状态：
- `DismissValue.Default`：初始状态
- `DismissValue.DismissedToEnd`：右滑完成
- `DismissValue.DismissedToStart`：左滑完成
