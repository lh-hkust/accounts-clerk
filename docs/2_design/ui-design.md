## 一、设计原则

### 1.1 核心原则

| 原则        | 说明                |
| --------- | ----------------- |
| **安全性优先** | 数据库加密，APP启动需密码/指纹验证 |
| **简洁高效**  | 清晰的信息层级，减少操作步骤    |
| **一致性**   | 统一的视觉风格和交互模式      |
| **可访问性**  | 支持不同屏幕尺寸和辅助功能     |
| **反馈及时**  | 操作后立即给出视觉反馈       |

### 1.2 设计目标

- 帮助用户快速了解身份标识变更的影响范围
- 提供直观的预警提醒机制
- 简化账户和身份标识的管理流程
- 保护用户隐私和数据安全

## 五、交互逻辑

### 5.1 渠道查看流程

```
用户打开APP
    → 弹出验证弹窗（指纹/密码）
    → 验证成功
    → 数据库解锁
    → 渠道值全文显示
```

### 5.2 设置到期提醒流程

```
用户点击"设置到期提醒"
    → 进入设置到期提醒页
    → 用户选择日期
    → 用户输入原因
    → 系统显示受影响账号
    → 用户确认
    → 创建到期计划
    → 创建提醒记录
    → 返回渠道详情页
```

### 5.3 提醒处理流程

```
用户查看提醒列表
    → 用户点击提醒卡片
    → 进入影响范围页
    → 显示受影响账号
    → 用户点击"标记已处理"
    → 标记提醒为已处理
    → 返回提醒列表
```

### 5.4 搜索流程

```
用户输入搜索关键词
    → 系统匹配渠道值
    → 系统匹配账号名/标识
    → 返回匹配的渠道/账号列表
    → 用户选择结果
    → 进入详情页
```

## 六、设备适配

### 6.1 响应式布局

| 设备类型   | 屏幕尺寸          | 布局策略      |
| ------ | ------------- | --------- |
| **手机** | < 600dp       | 单列布局，底部导航 |
| **平板** | 600dp - 900dp | 双列布局，侧边导航 |
| **大屏** | > 900dp       | 三列布局，侧边导航 |

### 6.2 适配原则

1. **弹性布局**：使用 ConstraintLayout 或 Flexbox
2. **可缩放字体**：使用 sp 单位
3. **响应式图片**：根据屏幕密度加载合适尺寸
4. **触摸目标**：最小触摸区域 48dp × 48dp

### 7.1 用语设计原则概要

| 原则 | 说明 | 示例 |
|------|------|------|
| **贴近用户心智** | 使用用户日常表达，而非技术术语 | "验证渠道"而非"身份标识" |
| **参考业界惯例** | 借鉴同类产品的成熟用语 | "账号"与1Password/Bitwarden一致 |
| **保持一致性** | 同一概念在所有界面使用统一用语 | 所有页面统一用"验证渠道" |
| **避免歧义** | 不用可能引起误解的专业术语 | "凭证"易与密码混淆，改用"渠道" |
| **正向表达** | 优先传达"解决方案"而非"问题描述" | "到期提醒"而非"停用预警" |

### 7.2 状态用语速查

| 领域状态 | UI用语 | 颜色 | 图标 |
|------|------|------|------|
| ACTIVE | 正常使用 | 绿色 (#22c55e) | ✓ |
| PENDING_DEACTIVATION | 即将到期 | 黄色 (#eab308) | ⏳ |
| DEACTIVATED / INVALIDATED | 已失效 | 红色 (#ef4444) | ✗ |
| FROZEN | 已冻结 | 红色 (#ef4444) | ❄️ |
| LOST | 已丢失 | 灰色 (#6b7280) | 💔 |
| ARCHIVED | 已归档 | 灰色 (#6b7280) | 📁 |

> 详细用语对照表和决策依据请查阅 [UI词汇表](../0_meta/ui-glossary.md) 和 [UI用语规范](../0_meta/ui-terminology-guide.md)
