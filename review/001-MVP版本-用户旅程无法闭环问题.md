

## 用户旅程无法闭环的问题清单

### 一、验证渠道管理

| 问题                           | 规格状态         | 缺失内容                                               |
| ------------------------------ | ---------------- | ------------------------------------------------------ |
| **编辑验证渠道**               | ❌ 未定义         | 点击已添加的渠道后能否修改？修改什么字段？值能改吗？   |
| **直接删除渠道（有绑定账号）** | ⚠️ 部分定义       | spec说"请先解绑"，但无解绑操作入口设计，解绑流程未闭环 |
| **停用计划选择过去时间**       | ✅ 已定义（拒绝） | spec明确禁止，但UI无错误提示样式设计                   |
| **停用计划修改流程**           | ✅ 已定义         | spec有场景，但原型无"修改计划"页面/入口                |

### 二、账号管理

| 问题                          | 规格状态         | 缺失内容                                                     |
| ----------------------------- | ---------------- | ------------------------------------------------------------ |
| **新增账号选择验证渠道交互**  | ⚠️ 部分定义       | spec说"绑定标识"，但无交互流程：是弹窗选择？页面跳转？搜索怎么触发？ |
| **选择绑定用途交互**          | ✅ 已定义用途列表 | 但无选择UI：多选？单选？checkbox？chip？                     |
| **编辑账号信息**              | ❌ 未定义         | accountName、accountIdentifier能否修改？修改后唯一性校验？   |
| **删除账号（有绑定渠道）**    | ⚠️ 部分定义       | spec无明确规则：自动解绑？阻止删除？确认弹窗内容？           |
| **账号卡片交互（长按/右滑）** | ❌ 未定义         | 无任何手势交互设计，主流App标配功能缺失                      |

### 三、数据导入导出

| 问题                 | 规格状态   | 缺失内容                                                     |
| -------------------- | ---------- | ------------------------------------------------------------ |
| **导入文件选择**     | ❌ 未定义   | 调用系统文件选择器？需要什么权限？Android 11+ Scoped Storage适配？ |
| **导出位置选择**     | ❌ 未定义   | 导出到哪里？Downloads目录？用户自选位置？SAF接口？           |
| **导入成功后的提示** | ⚠️ 部分定义 | spec说"显示结果摘要"，但无UI样式设计                         |
| **导出成功后的提示** | ❌ 未定义   | Toast？Dialog？是否显示文件路径？                            |
| **加密导出交互**     | ⚠️ 部分定义 | spec说"可设置导出密码"，但无：密码输入界面、加密格式、扩展名、解密说明 |
| **导入模式选择**     | ⚠️ 部分定义 | spec列出三种模式，但无选择界面设计                           |

### 四、设置与安全

| 问题                 | 规格状态       | 缺失内容                                                     |
| -------------------- | -------------- | ------------------------------------------------------------ |
| **应用启动验证流程** | ⚠️ 提及但未闭环 | ui-design说"密码/指纹验证"，但无验证界面设计、失败处理、生物识别fallback |
| **清空数据确认方式** | ⚠️ 部分定义     | spec说"输入CLEAR"，但UI无输入框设计                          |
| **设置密码流程**     | ⚠️ 部分定义     | spec说"最少6字符"，但无：二次确认、强度提示、密码修改入口    |

### 五、渠道卡片交互

| 问题                 | 规格状态 | 缺失内容                       |
| -------------------- | -------- | ------------------------------ |
| **长按卡片显示菜单** | ❌ 未定义 | 编辑、删除、设置到期提醒子菜单 |
| **右滑卡片快捷操作** | ❌ 未定义 | 删除、标记处理等快捷手势       |
| **点击渠道卡片导航** | ✅ 已定义 | 进入详情页，已实现             |

### 六、其他用户旅程断点

| 问题                      | 规格状态     | 缺失内容                                                     |
| ------------------------- | ------------ | ------------------------------------------------------------ |
| **批量更换验证渠道**      | ⚠️ 仅按钮存在 | 详情页有按钮，但无操作流程设计：目标渠道怎么选？逐个换还是一键换？ |

| **欢迎页到主页流程**      | ⚠️ 部分定义   | 原型有欢迎页，但无"已有账户？登录"逻辑（单机版无登录）       |
| **应用平台自定义添加**    | ❌ 未定义     | proposal说"允许用户自定义添加应用"，但无入口和流程           |

### 优先级建议（按用户旅程影响程度）

**P0 - 核心功能无法使用**：

1. 新增账号绑定验证渠道交互流程
2. 导入/导出文件选择流程
3. 编辑账号信息能力

**P1 - 主流交互缺失**： 4. 长按/右滑卡片快捷菜单 5. 删除账号/渠道时的解绑处理逻辑 6. 加密导出的完整交互

**P2 - 体验优化**： 7. 批量更换验证渠道流程 8. 自定义添加应用平台 9. 密码设置/修改完整流程
