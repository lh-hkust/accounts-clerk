

## 用户旅程无法闭环的问题处理意见

## 新增概念分析

### P0级缺失（必须纳入）

| 缺失事项                     | 新增概念/领域对象                       | 影响范围                     |
| ---------------------------- | --------------------------------------- | ---------------------------- |
| **新增账号绑定验证渠道交互** | 无新概念，是UI交互流程缺失              | 需补充交互设计规格           |
| **导入/导出文件选择流程**    | `ImportSession`、`ExportSession` 值对象 | 导入导出流程需要会话状态管理 |
| **编辑账号信息**             | 无新概念，已有属性可编辑                | 需补充编辑操作规格           |

### P1级缺失（建议纳入）

| 缺失事项                      | 新增概念/领域对象                                           | 影响范围           |
| ----------------------------- | ----------------------------------------------------------- | ------------------ |
| **长按/右滑卡片快捷菜单**     | `QuickAction` 值对象、`CardGesture` 交互模式                | 需补充UI交互规范   |
| **删除账号/渠道时的解绑处理** | `UnbindStrategy` 枚举（CASCADE/BLOCK/MANUAL）               | 需补充删除策略规格 |
| **加密导出的完整交互**        | `ExportEncryptionConfig` 值对象、`EncryptedExportFile` 实体 | 需补充加密导出规格 |

### P2级缺失（后续规划）

| 缺失事项                  | 新增概念/领域对象                              | 影响范围           |
| ------------------------- | ---------------------------------------------- | ------------------ |
| **批量更换验证渠道流程**  | `BatchBindingOperation` 聚合                   | 新聚合，V2.0规划   |
| **自定义添加应用平台**    | `CustomApplication` 实体                       | 需补充应用管理规格 |
| **密码设置/修改完整流程** | `AccessPassword` 值对象、`PasswordPolicy` 配置 | 需补充安全设置规格 |



## UI交互流程详细设计

### 1. 数据导入导出交互流程

**非安全模式（导出JSON）**

```
用户点击"导出JSON"
    → 弹出安全警告对话框
        → 标题："安全警告"
        → 内容："导出为明文JSON文件，数据无加密保护。
                  任何人均可查看文件内容，存在隐私泄露风险。
                  建议使用「安全导出」保护您的数据。"
        → 按钮："使用安全导出"（主按钮）、"继续导出JSON"（次按钮）
    → 若用户点击"使用安全导出"
        → 切换到安全导出流程
    → 若用户点击"继续导出JSON"
        → 用户勾选"我已了解风险"确认框
        → 点击"确认导出"
    → 调用SAF选择保存位置
        → 默认文件名：hermes_export_YYYYMMDD.json
    → 导出进度对话框
    → 导出完成
        → 成功提示："导出完成！文件：<路径>"
        → 警告提醒："请妥善保管此文件，避免上传到云端或分享给他人"
```

**安全模式（安全导出）**

```
用户点击"安全导出"
    → 弹出密码设置对话框
        → 标题："设置导出密码"
        → 说明："为导出文件设置密码，仅本应用可打开。
                  若不设置密码，文件仍加密但任何拥有本应用的人都能打开。"
        → 密码输入框（可选）
        → 确认密码框（若输入了密码）
        → 密码强度提示："至少6个字符"
        → 若用户未输入密码：
            → 显示警告："⚠️ 未设置密码，文件可被任何拥有本应用的人打开。请勿分享或上传到云端。"
            → 用户勾选"我已了解风险"
        → 按钮："确认导出"（主按钮）、"取消"（次按钮）
    → 用户点击"确认导出"
    → 调用SAF选择保存位置
        → 默认文件名：hermes_export_YYYYMMDD.hexport
    → 导出进度对话框
    → 导出完成
        → 成功提示："导出完成！文件已加密保存：<路径>"
```

**导入流程**

```
用户点击"导入数据"
    → 调用SAF选择文件
        → 文件过滤器：.hexport, .json
    → 解析文件类型
        → 若 .hexport（加密文件）：
            → 解析文件头KDF模式
            → 若 KDF=0x01（密码模式）：
                → 弹出密码输入对话框
                → 用户输入密码
                → 尝试解密
                → 若解密失败："密码错误或文件损坏"
            → 若 KDF=0x02（无密码模式）：
                → 自动尝试解密
                → 若失败："文件损坏或非本应用导出"
        → 若 .json（明文文件）：
            → 直接解析JSON
    → 解析成功，显示导入预览
        → 数据摘要：验证渠道 X个、账号 Y个、绑定关系 Z个
        → 冲突检测结果（如有重复数据）
    → 选择导入模式
        → 选项卡片："合并"（默认）、"覆盖"、"跳过重复"
        → 各模式说明：
            - 合并：新增不存在数据，保留已有数据
            - 覆盖：新增不存在数据，替换已有数据
            - 跳过重复：仅导入不存在数据，跳过所有重复
    → 用户点击"开始导入"
    → 导入进度对话框
    → 导入完成
        → 成功提示："导入完成！新增 X条，更新 Y条，跳过 Z条"
```

### 2.卡片快捷交互模式设计

**设计原则**

基于 Material Design 和主流 Android 应用实践，四种手势承载不同层级操作：

| 手势       | 交互层级   | 操作类型                                                     |
| ---------- | ---------- | ------------------------------------------------------------ |
| **点击**   | 详情查看   | 进入详情页，查看完整信息                                     |
| **长按**   | 快捷菜单   | 弹出操作菜单（编辑、删除等）                                 |
| **右左滑** | 单操作快捷 | 显示快捷操作按钮（单一高频操作）                             |
| **左滑**   | 单操作快捷 | 显示快捷操作按钮（单一高频操作），考虑到右手操作的用户更多，优先右滑 |

**验证渠道卡片交互**

```
┌─────────────────────────────────────────────────────────────┐
│ [图标] 13812345678                          [状态徽章]      │
│        正常使用 · 绑定5个账号                                 │

点击 → 进入「影响范围」详情页

左滑 → 显示「标记已处理」按钮（仅预警状态可见）
       正常状态：无操作

    → 用户需点击确认按钮, 才能删除, 点击取消则不删除

**预警卡片交互**

```

图标说明：
- HIGH（紧急）：红色圆形感叹号 Icons.Filled.Error
- MEDIUM（建议）：黄色三角形警告 Icons.Filled.Warning
- LOW（可稍后）：蓝色圆形信息 Icons.Filled.Info

点击 → 进入「影响范围」详情页（标识详情）

长按 → 弹出菜单：
       ├─ 标记已处理
       ├─ 设置到期提醒（若未设置）

右滑 → 显示「标记已处理」按钮（高频操作）

左滑 → 无操作
```

#### 预警级别视觉对照表

| 级别   | 图标                                 | 颜色              | 图标背景       |
| ------ | ------------------------------------ | ----------------- | -------------- |
| HIGH   | `Icons.Filled.Error`（圆形感叹号）   | #ef4444（危险色） | 红色半透明背景 |
| MEDIUM | `Icons.Filled.Warning`（三角形警告） | #eab308（警告色） | 黄色半透明背景 |
| LOW    | `Icons.Filled.Info`（圆形信息i）     | #3b82f6（主色）   | 蓝色半透明背景 |

#### 滑动手势优先级规则

**设计原则**：

- **右滑**承载最高频/最优先操作（用户最常用）
- **左滑**承载次要操作或无操作
- 若仅需一个滑动操作，则仅用右滑，左滑无操作

### 3.新增账号绑定验证渠道交互流程

**场景背景**

用户在「添加账号」页面，需要选择应用、填写账号信息，并绑定验证渠道。当前设计缺失绑定渠道的完整交互流程。

**新增账号完整流程**

```
用户点击"添加账号"
    → 进入「添加账号」页
    → 页面结构：
        ├─ 选择应用（图标网格）
        ├─ 账号ID输入框（必填）
        ├─ 昵称输入框（可选）
        └─ 绑定验证渠道区域
    → 绑定验证渠道交互流程：

**绑定验证渠道交互流程**

```
用户进入「绑定验证渠道」区域
    → 显示可用验证渠道列表（从数据库加载）
    → 若无可用渠道：
        → 提示："暂无可用验证渠道，请先添加验证渠道"
        → 提供"添加验证渠道"按钮（跳转到添加渠道页）
    → 若有可用渠道：
        → 显示渠道列表卡片
        → 点击渠道卡片 → 选中该渠道（高亮边框 + 勾选图标）
        → 再次点击已选中渠道 → 弹出「用途选择」对话框

用途选择规则：
- 多选模式（checkbox/chip）
- 至少选择一项才能确认
- LOGIN（登录）默认选中
- 点击"确认"后用途生效，卡片显示用途标签
```

```

## P2缺失事项概述及领域模型更新建议

### 一、P2缺失事项（后续规划V2.0）

#### 1. 批量更换验证渠道

**缺失场景**：用户需要将多个账号的验证渠道从旧手机号批量更换到新手机号

**建议设计**：



```
入口：影响范围页 → 点击「批量更换渠道」按钮

流程：
    → 显示当前渠道：13812345678（即将到期）
    → 显示受影响账号列表（14个）
    → 选择新渠道：13900001111
    → 预览更换结果：
        - 微博（登录、验证） → 13900001111
        - 淘宝（验证） → 13900001111
        - 微信（登录、验证） → 13900001111
        ...
    → 确认批量更换
    → 执行更换（逐个处理）
    → 显示结果：成功X个，失败Y个

领域对象建议：
- BatchBindingOperation（批量绑定操作聚合）
- BatchOperationItem（批量操作项实体）
- BatchOperationStatus（批量操作状态值对象）
```

#### 2. 自定义添加应用平台

**缺失场景**：用户需要添加不在预置清单中的应用

**建议设计**：



```
入口：添加账号页 → 应用选择区域 → 点击「自定义应用」按钮

流程：
    → 弹出添加自定义应用对话框
    → 输入应用名称（必填）
    → 选择应用分类（社交/金融/购物/其他）
    → 上传应用图标（可选，默认使用首字母图标）
    → 保存自定义应用
    → 新应用出现在应用选择列表中

领域对象建议：
- CustomApplication（自定义应用实体，继承Application聚合）
- ApplicationCategory（应用分类值对象）
```

#### 3. 密码设置/修改完整流程

**缺失场景**：用户设置应用访问密码，或修改已设置的密码

**建议设计**：



```
入口：设置页 → 隐私安全 → 访问密码

首次设置密码：
    → 输入新密码（至少6字符）
    → 确认新密码
    → 密码强度提示
    → 设置成功 → 显示密码已启用状态

修改密码：
    → 输入当前密码（验证）
    → 输入新密码
    → 确认新密码
    → 修改成功

关闭密码保护：
    → 输入当前密码（验证）
    → 警告："关闭密码保护后，任何人都能打开本应用查看数据"
    → 用户确认后关闭

领域对象建议：
- AccessPassword（访问密码值对象）
- PasswordPolicy（密码策略配置）
- PasswordStrength（密码强度值对象）
```

#### 4. 应用启动验证流程

**缺失场景**：用户打开应用时需要密码/指纹验证

**建议设计**：



```
启动验证流程：
    → 应用启动
    → 检查是否设置密码
    → 若已设置密码：
        → 显示验证界面
        → 选项：密码输入框 + 指纹/面容按钮

### 三、数据表更新建议

#### 无需新增表（P0/P1）

- `export_session`、`import_session`仅为临时状态，不持久化
- 已有的 `export_record`、`import_record` 表用于审计记录

#### 新增表（P2规划）

**batch_binding_operation表**：



```sql
CREATE TABLE batch_binding_operation (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    old_identifier_id INTEGER NOT NULL,
    new_identifier_id INTEGER NOT NULL,
    operation_type TEXT NOT NULL,    -- BATCH_SWITCH
    status TEXT NOT NULL,            -- PENDING/IN_PROGRESS/COMPLETED/FAILED
    total_items INTEGER NOT NULL,
    success_count INTEGER DEFAULT 0,
    failed_count INTEGER DEFAULT 0,
    created_at TEXT NOT NULL,
    completed_at TEXT,
    
    FOREIGN KEY (old_identifier_id) REFERENCES identity_identifier(id),
    FOREIGN KEY (new_identifier_id) REFERENCES identity_identifier(id)
);
```

**custom_application表**：



```sql
CREATE TABLE custom_application (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    category TEXT NOT NULL,
    icon_url TEXT,
    created_by_user INTEGER NOT NULL DEFAULT 1,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```
